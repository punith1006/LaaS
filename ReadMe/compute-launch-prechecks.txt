SECTION 4: PORT ALLOCATION LOGIC
Port Ranges
plaintext
NGINX_PORT_MIN, NGINX_PORT_MAX = 8100, 8199 (100 possible ports)
DISPLAY_MIN, DISPLAY_MAX = 20, 99                (80 possible displays)
ALLOCATABLE_CORES = list(range(2, 16))           (14 cores; 0-1 reserved for OS)
Port Allocation Process
Acquire file lock on /tmp/laas-resource-lock (exclusive fcntl lock)
Query running containers for currently used ports via docker inspect --format {{json .NetworkSettings.Ports}}
Iterate 8100-8199:
Skip if port already used by another container
Skip if port in RESERVED_PORTS (monitoring stack)
Skip if derived ports (nginx+1000, nginx+11000) conflict with RESERVED_PORTS
Return first available port
Allocate display (20-99) same process
Allocate CPU cores (contiguous block of needed size from 2-15)
Release lock
Critical: Port allocation is not atomic with container creation — allocation happens under lock, but container launch is outside lock. Race conditions possible if multiple orchestration requests arrive simultaneously.Evidence: app.py lines 356-408, 706-739
Example Port Allocation
Nginx port allocated: 8101
Selkies port (derived): 8101 + 1000 = 9101
Metrics port (derived): 8101 + 11000 = 19101
Display: :23
Container listens internally: nginx on :8080, selkies on :9080, metrics on :19080
Host port mapping: 8101:8080, (selkies not published), 19101:19080

Trigger: storage_type == "stateful" in launch request
Mount point: {NFS_MOUNT_ROOT}/{storage_uid} → /home/ubuntu inside container
Example: /mnt/nfs/users/u_1234567890abcdef12345678:/home/ubuntu
Storage Validation Steps (at launch time)
Parse storage_uid: Must match pattern ^u_[0-9a-f]{24}$ (25 characters: "u" + 24 hex)
Check mount exists: os.path.exists({NFS_MOUNT_ROOT}/{storage_uid})
Verify is directory: os.path.isdir(...)
Test writability: Create/delete .laas-write-test-{session_id[:8]} file
Emit event with mount path if success; fail session if any check fails
Evidence: app.py lines 741-770
Ephemeral Session Storage
Trigger: storage_type == "ephemeral"
Mount: None (container has no persistent /home/ubuntu)
Effect: Container uses tmpfs only; data lost on stop
NFS vs ZFS
Production uses: NFS-mounted ZFS subdatasets
Per-user isolation: Each user gets /datapool/users/{storage_uid} via NFS at /mnt/nfs/users/{storage_uid}
Quota enforcement: Enforced by ZFS (not app.py; separate provisioning service)
app.py role: Validate mount exists and is writable before container launch

SECTION 11: COMMON FAILURE MODES & DIAGNOSTICS
Symptom: "launch timed out after 120 seconds"
Root Cause: Bridge network broken (nginx responds inside container but not externally)
Diagnostic:
bash
# Inside container
docker exec <container_name> curl -v http://127.0.0.1:8080/

# From host (should fail if bridge broken)
curl -v http://127.0.0.1:{nginx_port}/

# Check iptables
sudo iptables -L DOCKER-CT -n -v | head -20  # Look for 0 packets

# Check bridge route
ip route show | grep 172.31
Fix: Recreate bridge network (see Section 1)
Symptom: "No available ports in range 8100-8199"
Root Cause: All 100 ports allocated to other containers
Diagnostic:
bash
docker ps -a --filter "label=laas.session_id"
curl http://127.0.0.1:9998/resources  # Check port utilization
Fix: Clean up old containers or use larger port range
Symptom: Container exits immediately after start
Diagnostic:
bash
docker logs <container_name>
docker inspect <container_name> | grep -i error
Common Causes:
Missing /etc/laas/ files (supervisord fails to start)
HAMi library missing (LD_PRELOAD fails)
lxcfs not running (proc files not available)
Symptom: Container has no GPU access
Diagnostic:
bash
docker exec <container_name> nvidia-smi
Common Causes:
NVIDIA Container Toolkit not installed
--gpus all flag missing (check docker create command)
CUDA_VISIBLE_DEVICES not set in container
