## Critical Findings: Database Schema Review

### 🔴 CRITICAL GAPS (Must Address Before POC)

#### 1. **No Session Management Tables**
The current Prisma schema has **zero tables** for the core platform functionality:
- No `sessions` table (container lifecycle, WebRTC streaming, node assignment)
- No `bookings` table (scheduled reservations)
- No `compute_configs` table (Starter/Pro/Max tiers)
- No `nodes` table (infrastructure tracking)

**Impact:** The platform cannot track user sessions at all. This is the absolute core of LaaS.

#### 2. **Storage Model Mismatch**
- **Current schema:** Stores `storageUid`, `storageProvisioningStatus`, `storageProvisioningError`, `storageProvisionedAt` directly on `User`
- **Plan specifies:** Separate `user_storage_volumes` table with:
  - `zfs_dataset_path`, `nfs_export_path`, `container_mount_path`
  - `quota_bytes` (15GB), `used_bytes`, `used_bytes_updated_at`
  - `status` ('provisioning'/'active'/'wiping'/'wiped'/'error')
  - `os_choice` for multi-OS support

**Issue:** The current design cannot support OS switching (a key requirement: "permanently wipes 15GB, requires explicit DELETE confirmation").

#### 3. **Missing University/Multi-tenancy Tables**
- No `universities` table (only `organizations`)
- No `departments` table
- No `user_groups` table (semester, batch, research groups)
- No `university_idp_configs` table (SAML/OIDC/LDAP configs per university)

**Impact:** Cannot federate with university SSO (Keycloak identity brokering per university).

#### 4. **Access Control for Public Users**
Per project context:
> "for public users, only ephemeral instances either with notebook mode or CLI (depending on the config they choose)"

- No `compute_config_access` table to enforce: public users → ephemeral only, university members → stateful + ephemeral

Project requirement: "Ubuntu 22.04 LTS for now! and I'll provide other distros later"
- Current schema has no `os_choice` field
- Plan includes `os_choice VARCHAR(32)` on users table

Per project context, billing is core functionality:

**Note:** For POC, you need at least basic wallet tracking even if full billing comes later.

### 🟡 TECHNICAL ISSUES

#### 8. **Enum Storage as VARCHAR (Plan Convention)**
Plan wisely uses VARCHAR for enums instead of PostgreSQL enum types. **However, the current Prisma schema uses raw strings without validation.**

**Recommendation:** Add application-level enum validation or use Prisma's native enum types for better type safety:

```prisma
enum AuthType {
  LOCAL
  PUBLIC_OAUTH
  UNIVERSITY_SSO
}

enum StorageProvisioningStatus {
  PENDING
  PROVISIONING
  PROVISIONED
  FAILED
}
```

#### 9. **No Index Recommendations**
High-cardinality fields that need indexes:
- `users.keycloak_sub` ✓ (already unique)
- `users.storage_uid` ✓ (already unique)
- `sessions.user_id` - needs index
- `sessions.node_id` - needs index
- `sessions.status` - needs index (for filtering active sessions)
- `bookings.user_id` - needs index
- `bookings.scheduled_start_at` - needs index (for conflict detection)
- `login_history.user_id` - needs index
- `login_history.created_at` - needs index (for partition pruning)

#### 10. **Missing `session_type` Field Differentiation**
The plan has session types:
- `stateful_desktop` / `ephemeral_jupyter` / `ephemeral_codeserver` / `ephemeral_cli`

But no table tracks which type each user can access. This is critical for:
- Public users: ephemeral ONLY
- University members: stateful + ephemeral

#### 11. **`os_switch_history` Missing**
When a user switches OS, the old storage must be wiped and a new one created. Need audit trail:
- Who approved (if admin-initiated)
- What was the confirmation text ("DELETE")
- IP at time of switch
- Old/new volume IDs

2. **UUID PKs everywhere** - Consistent, non-guessable identifiers
3. **Audit columns** - created_at, updated_at, created_by
4. **Organization-level multi-tenancy** - Right abstraction level
5. **Refresh token with device info** - Good for security tracking
6. **Separate `user_org_roles` junction** - Allows users in multiple orgs with different roles
