Complete Installation & Setup Guide
Bare Metal Ubuntu 22.04 → Fully Validated GPU Session Node
March 2026  |  Version 1.0  |  Based on POC Runbook v2.0




About this Guide: This is the complete, as-executed installation guide for a LaaS GPU session node. Every command has been run and validated on the POC machine. Commands are presented in exact execution order. All deviations from standard expectations are flagged. This guide takes you from a fresh Ubuntu 22.04 installation to a fully running, multi-session GPU desktop node.

Hardware Reference

⚠ WARNING: RTX 5090 is Blackwell (sm_100). All validation was on RTX 4090 (Ada, sm_89). Change CUDA_NVRTC_ARCH=89 to CUDA_NVRTC_ARCH=100 in all container launch commands on production nodes. Re-validate HAMi enforcement and NVENC encoding on first production node before fleet deployment.

  STEP 5    HAMi-Core Build (VRAM + SM Enforcement)

HAMi-core compiles to libvgpu.so — a shared library that intercepts CUDA API calls to enforce per-container VRAM and SM limits. No kernel module, no daemon. Pure LD_PRELOAD interception.

5.1 — Build libvgpu.so

cd ~
git clone https://github.com/Project-HAMi/HAMi-core.git
cd HAMi-core
mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
make -j$(nproc)
 
# Verify build
ls -la libvgpu.so
# Expected: ~418784 bytes
 
# Install to correct path
sudo cp libvgpu.so /usr/lib/libvgpu.so
sudo ldconfig
ls -la /usr/lib/libvgpu.so

🔴 CRITICAL: Install path MUST be /usr/lib/libvgpu.so — NOT /usr/local/lib/. Wrong path causes silent failures where HAMi loads but LD_PRELOAD is never resolved inside containers.

5.2 — Backup (before any future patching)

sudo cp /usr/lib/libvgpu.so /usr/lib/libvgpu.so.backup
ls -la /usr/lib/libvgpu.so.backup

ℹ INFO: If you later build from the patched fork (loiht2/HAMi-core-fix-memory) for improved nvidia-smi NVML interception, restore from this backup if anything breaks.

  STEP 6    CUDA MPS Daemon (Compute Sharing)
MPS (Multi-Process Service) allows multiple containers to share the GPU's SMs concurrently. Required for SM% limits to be meaningful. Must run as a systemd service on the host.


sudo tee /etc/systemd/system/cuda-mps.service > /dev/null << 'EOF'
[Unit]
Description=CUDA MPS Control Daemon
After=nvidia-persistenced.service
 
[Service]
Type=forking
Environment=CUDA_VISIBLE_DEVICES=0
Environment=CUDA_MPS_PIPE_DIRECTORY=/tmp/nvidia-mps
Environment=CUDA_MPS_LOG_DIRECTORY=/tmp/nvidia-log
ExecStartPre=/usr/bin/nvidia-smi -pm 1
ExecStart=/usr/bin/nvidia-cuda-mps-control -d
ExecStop=/bin/bash -c "echo quit | /usr/bin/nvidia-cuda-mps-control"
Restart=on-failure
RestartSec=10
 
[Install]
WantedBy=multi-user.target
EOF
 
sudo systemctl daemon-reload
sudo systemctl enable --now cuda-mps
sudo systemctl status cuda-mps
# Expected: active (running)

ℹ INFO: echo get_server_list | nvidia-cuda-mps-control returning empty is NORMAL. MPS servers spawn on-demand when a CUDA process connects. Not a failure.

  STEP 8    lxcfs (Resource Visibility Spoofing)
lxcfs is a FUSE filesystem that intercepts /proc reads and returns container-cgroup-aware values. Without it, containers see the host's full 64GB RAM and 32-core CPU in free -h, KDE System Settings, and all tools that read /proc.


sudo apt install -y lxcfs
sudo systemctl enable --now lxcfs
sudo systemctl status lxcfs
# Expected: active (running)
 
# Verify all required files are available
ls /var/lib/lxcfs/proc/
# Must see: cpuinfo meminfo stat uptime loadavg diskstats swaps
 
ls /var/lib/lxcfs/sys/devices/system/cpu/
# Must see: cpu0 cpu1 ... online possible present offline isolated

ℹ INFO: lxcfs must be running on the host before launching any containers. Add to all node startup checks. If lxcfs is not running, containers will show host specs to users.


  STEP 9    Build fake_sysconf.so (RAM Fix for KDE)
KDE reads RAM via sysconf(_SC_PHYS_PAGES) syscall — not /proc/meminfo. lxcfs cannot intercept syscalls, only file reads. This tiny LD_PRELOAD library intercepts the sysconf call and returns the container-correct RAM value.


cat > /tmp/fake_sysconf.c << 'EOF'
#define _GNU_SOURCE
#include <unistd.h>
#include <dlfcn.h>
#include <stdlib.h>
#include <string.h>
 
// Reads CONTAINER_MEMORY_LIMIT_BYTES env var set at container launch
// Returns correct page count matching the container's RAM limit
long sysconf(int name) {
    static long (*real_sysconf)(int) = NULL;
    if (!real_sysconf)
        real_sysconf = dlsym(RTLD_NEXT, "sysconf");
 
    if (name == _SC_PHYS_PAGES || name == _SC_AVPHYS_PAGES) {
        const char *limit = getenv("CONTAINER_MEMORY_LIMIT_BYTES");
        if (limit) {
            long page_size = real_sysconf(_SC_PAGESIZE);
            long limit_bytes = atol(limit);
            long pages = limit_bytes / page_size;
            if (name == _SC_AVPHYS_PAGES)
                return (long)(pages * 0.85);
            return pages;
        }
    }
    return real_sysconf(name);
}
EOF
 
# Compile
gcc -shared -fPIC -o /tmp/fake_sysconf.so /tmp/fake_sysconf.c -ldl
 
# Install
sudo cp /tmp/fake_sysconf.so /usr/lib/fake_sysconf.so
 
# Verify
ls -la /usr/lib/fake_sysconf.so
# Expected: ~16K

ℹ INFO: CONTAINER_MEMORY_LIMIT_BYTES is passed as an env var at container launch time (e.g., 8589934592 for 8GB). The library reads this at runtime so no recompilation is needed per container.
🔴 CRITICAL: Do NOT add fake_sysconf.so to selkies-gstreamer's LD_PRELOAD — it causes exit 139 (SIGSEGV). Only load it in the entrypoint (for KDE) and terminal sessions (via bash.bashrc).

10.2 — nvidia-smi Wrapper

cat > /tmp/nvidia-smi-wrapper << 'WRAPPER_EOF'
#!/bin/bash
# Reads enforced VRAM limit from CUDA_DEVICE_MEMORY_LIMIT_0 env var
# Replaces total VRAM in nvidia-smi output with container's actual limit
 
get_vram_limit_mib() {
  if [ -n "$CUDA_DEVICE_MEMORY_LIMIT_0" ]; then
    VAL="${CUDA_DEVICE_MEMORY_LIMIT_0,,}"
    if [[ "$VAL" == *"m" ]]; then
      echo "${VAL//m/}"
    elif [[ "$VAL" == *"g" ]]; then
      echo $(( ${VAL//g/} * 1024 ))
    fi
    return
  fi
  python3 -c "
import struct
with open('/tmp/cudevshr.cache','rb') as f:
    d=f.read(4096)
print(struct.unpack_from('<Q',d,1600)[0]//(1024*1024))
" 2>/dev/null || echo "0"
}
 
# Pass through non-display commands unchanged
case "$*" in
  *"-pm"*|*"--persistence-mode"*|*"-pl"*|*"--power-limit"*|\
  *"--query-gpu"*|*"--format"*|*"-i "*|*"--id"*)
    exec env -u LD_PRELOAD /usr/bin/nvidia-smi.real "$@"
    ;;
esac
 
LIMIT_MIB=$(get_vram_limit_mib)
REAL_OUT=$(env -u LD_PRELOAD /usr/bin/nvidia-smi.real "$@" 2>&1)
 
if [ -n "$LIMIT_MIB" ] && [ "$LIMIT_MIB" -gt "0" ] 2>/dev/null; then
  echo "$REAL_OUT" | sed "s|/ [0-9]*MiB|/ ${LIMIT_MIB}MiB|g"
else
  echo "$REAL_OUT"
fi
WRAPPER_EOF
chmod +x /tmp/nvidia-smi-wrapper

10.6 — SM% Calculator Script

sudo tee /usr/local/bin/calc-sm-percent << 'EOF'
#!/bin/bash
# Usage: calc-sm-percent <container_vram_gb> <total_gpu_vram_gb>
# SM% = VRAM_allocated / total_VRAM * 100 (proportional allocation)
# RTX 4090 POC:  total=24   RTX 5090 Production: total=32
CONTAINER_VRAM=${1:?Usage: calc-sm-percent <container_vram_gb> <total_gpu_vram_gb>}
TOTAL_VRAM=${2:?Usage: calc-sm-percent <container_vram_gb> <total_gpu_vram_gb>}
PCT=$(( (CONTAINER_VRAM * 100) / TOTAL_VRAM ))
[ $PCT -lt 5 ] && PCT=5
echo $PCT
EOF
sudo chmod +x /usr/local/bin/calc-sm-percent
 
# Verify

10.8 — Static CPU Sys Files (per container)
lxcfs does not filter /sys/devices/system/cpu by cpuset. Static files with exact correct cpuset ranges are created and mounted per container. KDE reads these to report CPU count.

for i in 1 2 3 4; do
  CPUSET_START=$(( ($i-1)*4 ))
  CPUSET_END=$(( ($i-1)*4+3 ))
  CPUSET="${CPUSET_START}-${CPUSET_END}"
  mkdir -p /tmp/container-$i-cpu
  echo "$CPUSET" > /tmp/container-$i-cpu/online
  echo "$CPUSET" > /tmp/container-$i-cpu/present
  echo "$CPUSET" > /tmp/container-$i-cpu/possible
  echo ""         > /tmp/container-$i-cpu/offline
  echo ""         > /tmp/container-$i-cpu/isolated
  chmod 444 /tmp/container-$i-cpu/*
  echo "Container $i → cpuset: $CPUSET → $(cat /tmp/container-$i-cpu/online)"
done

Limitation | Cause | Mitigation
nvidia-smi shows full GPU VRAM | NVML queries kernel driver — bypasses all user-space interception | nvidia-smi wrapper reads CUDA_DEVICE_MEMORY_LIMIT_0 env var and replaces total VRAM in output
Memory bandwidth shared | Consumer GPU (no MIG support on RTX) | SM proportional allocation minimises contention. Disclosed in ToS as shared-GPU limitation
--network=host (POC) | WebRTC requires direct port access over Tailscale | Production: --network=bridge + Cloudflare Tunnel
df shows overlay + NFS paths | Kernel-level mounts always visible | Use abstract NFS paths with no user-identifying info
lxcfs /sys filtering | lxcfs doesn't filter /sys by cpuset | Static per-container CPU files mounted at /sys/devices/system/cpu
Container startup ~15-25s | KDE Plasma initialisation time | Pre-warm pool for production UX

Symptom | Cause | Fix
selkies-gstreamer exit 139 in logs | Both libvgpu.so AND fake_sysconf.so in selkies-gstreamer LD_PRELOAD | Remove fake_sysconf.so from selkies-gstreamer — libvgpu.so only
LD_PRELOAD not set in Konsole | Injected in profile.d instead of bash.bashrc | Ensure /etc/bash.bashrc has the injection block
VRAM shows 23.5GB in ctypes test | ctypes.CDLL uses RTLD_LOCAL — bypasses LD_PRELOAD | Use torch for VRAM tests. ctypes always shows physical GPU.
nvidia-smi segfault | LD_PRELOAD + NVML conflict | Ensure nvidia-smi wrapper strips LD_PRELOAD before calling real binary
KDE RAM still shows host value | CONTAINER_MEMORY_LIMIT_BYTES env var missing | Verify -e CONTAINER_MEMORY_LIMIT_BYTES=$MEM_BYTES in docker run
nproc shows 32 | --cpuset-cpus not in docker run | Add --cpuset-cpus=0-3 (or appropriate range) to docker run
KDE CPU shows host cores | /tmp/container-$i-cpu not mounted | Verify -v /tmp/container-$i-cpu:/sys/devices/system/cpu:ro in docker run
Container not accessible in browser | selkies-gstreamer in FATAL state | Check docker logs <name> --tail 30 — look for crash reason
HAMi not enforcing limits | libvgpu.so at wrong path | Must be /usr/lib/libvgpu.so — not /usr/local/lib/
